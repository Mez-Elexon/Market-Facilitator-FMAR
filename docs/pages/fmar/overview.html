Programme Overview

The Flexibility Market Asset Registration (FMAR) programme is developing the central registry for all flexible assets participating in GB flexibility markets. This is the Single Source of Truth for asset-specific data.

FMAR Hub Components

  • ARM (Asset Registry Module): Definitive central registry for all flexible assets
  • SPUM (Service Provider User Management): Central registry for all entities interacting with FMAR Hub
  • PSQM (Product Service Qualification Module): Manages qualification and verification status
  • GICM (Grid Information and Control Module): Manages grid-related operational status

Design Workflow

Our iterative design process follows this workflow:

  1. Artefact Publication: Architecture Team publishes vX.Y of an artefact
  2. Feedback via Issues: Stakeholders provide feedback using GitHub Issues
  3. Triage & Labelling: Issues are categorized and prioritized
  4. Discussion: Clarifications and broader points are addressed
  5. Internal Decision: Architecture Team decides on feedback
  6. Artefact Refinement: Changes are incorporated offline
  7. New Version Publication: Updated artefact is published
